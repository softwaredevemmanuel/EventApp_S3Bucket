About This Project

Case Study :

Event-XYX is a go-to ticketing platform where admins can create events and such events are available for users to view, reserve a spot and pay.
Task :
For an MVP, I Built and deployed, this product in Python, Django, HTML, CSS, JQuery, JavaScript :
  1. - It Has an admin Login (use 'admin' & 'admin' as email and password for the admin user)
  2. - Admin can create a new event.
  3. - Every event has a name, location & poster image attached to them.
  4. - Every event has an event date and a bookings end date (date when bookings would end)
  5. - A New event has a type : either be free or paid.
  6. - Free events have no space limits or price attached to them
  7. - Paid events have a price and space capacity.
  8. - All these are specified before an event is created.
  9. - Admin can see ALL EVENTS
  10. - Admin can edit each event : Admin can only edit, if the event date has not passed; Admin can change the end date for ticket booking and date of the event. Admin can't change the type. For Paid events, you can increase the space capacity. You can only reduce the space capacity if and if only the number of bookings have not reached the number you are reducing it to. You can cancel an event if and if only NOBODY has booked.
  11. - Admin can view each event and see basic insight like how many people have viewed, how many people have booked, for paid events : how much they have made, how many slots are left.
Frontend
  1. -Has a user landing page where all events are shown
  2. - User can filter events by type (paid/free) and can sort by date of event (asc/desc)
  3. - User can view an event : See its name, poster image, type (paid or free), event date, how many slots left (paid), how many days left for booking {since there is no login, every view of the event counts on the view analytics for the event}
  4. - Canceled events don't show up
  5. - A user search for any event using the search form
  6. - The events are paginated to render a maximum of 8 events per page and you can NEXT or PREV a page
  7. - User can book an event using just email address and name.
  8. - For booking, if email already exists for that particular event, it will display an error message
  9. - For Paid events, You can check how many people have booked before booking.
  10. - For every event, you are allowed to book before booking date is exceeded
  11. - A user confirmation email is sent with further details of the event
